# 项目实战 - 天气查询应用 PRD

## 项目概述
本项目是一个天气查询应用，旨在让用户快速了解当前天气、未来预报及相关气象指数。应用需支持通过城市名称搜索或自动定位（Geolocation API）获取天气信息，并允许用户收藏常用城市。项目将对接外部第三方 API（OpenWeatherMap）。

## 核心功能需求

1. **天气查询与搜索功能**
   - 允许用户在搜索框输入城市名称并查询。
   - 调用外部 API（OpenWeatherMap）获取数据。
   - 核心展示内容：
     - 当前天气：温度、天气状况（晴、多云、雨等）、最高/最低温度、AQI（空气质量）。
     - 小时预报：未来几小时的逐小时天气预报（带小图标和温度）。
     - 多天预报：未来几天的简要天气预报。
     - 详细数据：风力/风向、湿度、紫外线指数、日落时间、能见度、体感温度等。
   - **加载与错误处理**：查询过程中必须显示加载动画 (Loading)；查询失败时（如“找不到该城市”或网络错误），需给出友好的错误提示。

2. **自动位置定位**
   - 使用浏览器原生的 `Geolocation API` 自动获取用户的经纬度。
   - 根据经纬度自动发起 API 请求并展示当地天气。
   - 如果用户拒绝定位权限，或者浏览器不支持定位，则显示友好的降级提示，引导用户手动搜索城市。

3. **城市收藏功能**
   - 用户可以将查询过的城市添加为“收藏”或“关注”。
   - 收藏的城市列表保存在本地 `LocalStorage` 中。
   - 用户可以直接点击收藏列表中的城市快速发起查询。
   - 支持删除收藏的城市。

## 扩展思路 (后续迭代计划)
- **生动UI**：根据当前天气状况（雨天、雪天、晴天）动态改变应用背景，并添加对应的天气图标和动画效果。
- **空气质量指数扩展**：更详细地展示 AQI 和健康建议。
- **多城市对比**：允许在同一个面板中添加多个城市的微型卡片，方便同时对比。
- **天气预警**：展示来自气象部门的恶劣天气预警。
- **历史数据与趋势分析**：查询历史天气，用折线图分析气温趋势。
